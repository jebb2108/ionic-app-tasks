# Ionic React iOS App — Task Checklist

## 🎨 Design Phase

- [ ] **Define app concept** — описать идею приложения, целевую аудиторию, ключевые фичи
- [ ] **Compare 2-3 design concepts** — проанализировать варианты навигации (tabs/menu/hybrid)
- [ ] **Select best concept** — выбрать лучшую концепцию с обоснованием
- [ ] **Identify key screens** — определить все экраны приложения
- [ ] **iOS HIG compliance check** — проверить соответствие Human Interface Guidelines
- [ ] **UX pattern recommendations** — рекомендовать UX паттернт

## 🏗️ Architecture Phase

- [ ] **Define component tree** — спроектировать дерево компонентов
- [ ] **React Router setup** — настроить маршрутизацию (tabs, stack, nested)
- [ ] **State management** — выбрать и настроить Redux/Zustand/Context
- [ ] **Capacitor plugins list** — определить необходимые плагинт
- [ ] **Folder structure** — создать структуру проекта
- [ ] **iOS-specific considerations** — safe areas, notch, keyboard, status bar

## 💻 Development Phase

- [ ] **Project scaffolding** — `ionic start` with React template
- [ ] **TypeScript types** — настроить strict типт
- [ ] **Core pages** — создать все основные страницы
- [ ] **Shared components** — создать переиспользуемые компоненты
- [ ] **Services & hooks** — API сервисы, кастомные хуки
- [ ] **Capacitor integration** — подключить и настроить плагины
- [ ] **Error handling** — error boundaries, loading states
- [ ] **iOS layout fixes** — safe areas, notch, keyboard handling

## 🔍 Review Phase

- [ ] **TypeScript strict check** — проверить типы
- [ ] **Ionic best practices** — проверить соответствие Ionic стандартам
- [ ] **Code quality** — naming, structure, comments
- [ ] **iOS compatibility** — проверить iOS-specific UI
- [ ] **Security review** — проверить на уязвимости
- [ ] **Final verdict** — APPROVED / CHANGES REQUIRED
